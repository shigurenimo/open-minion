# ミニオン図鑑(種族カタログ)

正本: [`lib/engine/collection/species.ts`](../lib/engine/collection/species.ts) の `DEFAULT_MINION_SPECIES`。
月齢の計算は [`lib/engine/collection/moon.ts`](../lib/engine/collection/moon.ts)。

## 解決ルール

- 出現: `resolveSpecies(stats)` はカタログを先頭から順に評価し、最初に条件を満たした1体を「いま現れているミニオン」として返す。配列の順序 = 優先度。
- 発見: `evaluate()` はその瞬間条件を満たしている全種族を発見扱いにする(出現している1体だけではない)。恒常条件のレア(累計トークン系など)が上位で常に出現し続けても、下のコモンや別のレアの発見は妨げられない — 図鑑は必ずコンプ可能。
- レア・限定性の高い条件ほど上に置く。末尾の時間帯コモン5体は全時間をカバーする保証付きフォールバック — 必ず最後に置く(カスタムカタログでも全カバーのフォールバックがないと `resolveSpecies` は `Error` を返す)。
- 発見は `minion dex` 実行時に評価され、`~/.minion/collection.json` に初発見日時つきで永続化される。未発見の種族はCLI上 `???` 表示。
- カタログはDI可能: `new Minion({ species })` で丸ごと差し替え。`asset` は任意の不透明文字列(スプライトパス等)で、ライブラリは読まない。
- 時間限定でも「狙えば捕まえられる」条件にする。年1回のような取り逃すと長く待つ条件はコレクションの気力を削ぐので入れない(方針)。最長でも月1回(満月・新月・月末)、多くは毎日〜毎週狙える。

## コモン(時間帯フォールバック)

- `late-night` — しんやミニオン — 深夜 0時〜5時
- `morning` — あさミニオン — 朝 5時〜10時
- `day` — ひなたミニオン — 日中 10時〜17時
- `evening` — ゆうやけミニオン — 夕方 17時〜20時
- `night` — よふかしミニオン — 夜 20時〜24時

## 条件に使える材料(`StatsSnapshot`)

- `now` — 評価時刻の `Date`(曜日・日付・分・月齢などはここから導出)
- `hour` / `timeBucket` — ローカル時(`lateNight`/`morning`/`day`/`evening`/`night`)
- `currentConcurrentSessions` — いまの同時セッション数
- `maxConcurrentSessions` — 同時セッション数の最高記録
- `totalSessionsSeen` — 累計セッション数
- `uniqueProjectsSeen` — 異なるプロジェクト数
- `currentStreakDays` / `longestStreakDays` — 連続日数(現在/最長)
- `tokensTotal` — 累計トークン(minion導入以降の積算 — 7日バックフィル窓より古い履歴は含まない)
- `tokensToday` / `tokensThisWeek` — 当日/直近7日のトークン(UTC日付バケツ)

## 追加時の注意

- 取り逃しても近いうちに再挑戦できる条件だけにする(上記の方針)。梯子(2→3→5→10 のような段階)を作ると序盤から発見が続いて楽しい。
- 限定性の高い順に並べる(分単位 > 月1回 > 週1回 > 常時条件)。
- 月齢は平均朔望月のモジュロ近似で実際の満月と最大±1日ずれる — 判定の許容幅±1日はそれを吸収するためのもの。
- 条件は純関数(`stats` 以外を読まない)。`Date.now()` 直呼び禁止、時刻は `stats.now` から。
- 日付・曜日条件のテストはローカル時刻フィールドで `Date` を組む(タイムゾーン安全性、CLAUDE.md参照)。月齢条件のテストは絶対時刻(UTC ISO)でよい。
